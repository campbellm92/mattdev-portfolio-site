Il mio stack

08.10.24

Il mio stack lo chiamo MyHEN ed è composto da:

  • MySQL per il database
  • HBS (Handlebars) per il server-side rendering di HTML
  • Express per il routing
  • Node.js come runtime e per moduli rilevanti

Ho iniziato a usare questo stack soprattutto perché la mia traiettoria di apprendimento dello sviluppo ha privilegiato Node.js prima di React. Ora conosco React, ma ho ancora una predilezione per creare siti statici con questo stack. Forse cambierò idea quando avrò imparato meglio React; sicuramente questo stack non è così dinamico o efficiente come i framework frontend moderni. Tuttavia, credo che ci sono dei vantaggi nel realizzare progetti con questo stack:

  • Lo stack è una buona scelta per creare siti statici o piccoli se non c’è bisogno di includere molta interattività nell'app.
  • È uno stack valido per imparare il backend. Poiché Handlebars semplifica il lato client, hai più tempo per concentrarti sul backend.
  • Handlebars è relativamente facile da usare, soprattutto se hai un gusto per un'UI minimalista, come questo sito.
  • Con questo stack, e senza la comodità di un progetto con una toolchain moderna (JSX, state management, ecc.), ho avuto spesso la necessità di scrivere Vanilla JavaScript per gestire qualche aspetto del DOM. Questo sicuramente non è un modo moderno o efficiente per creare un'app, ma aiuta a capire meglio cosa succede "under the hood" e (come ho sentito dire) c'è ancora la possibilità di trovarsi in un'azienda in cui le novità dei framework frontend non sono ancora arrivate.
